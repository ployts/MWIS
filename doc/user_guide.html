User Guide

1. Deployment

Step 1

Excute 'compile.sh' for compilation. When the process is finished the binaries 'mwis' can be found in the root directory.

Step 2

Run the executable file 'mwis' with the following options and parameters.

Option Parameter Description
-r filepath Read the input file from the 'filepath'
-o filepath Write the output file to the 'filepath'
-w Choose how the algorithm run, reduce only or run the whole
-t timelimit Set the time limit in seconds. (Default: 1000s)
-h Print helps

2. I/O Format

Input File Format

A graph G=(V,E)G=(V,E)G=(V,E) with nnn vertices and mmm edges is stored in a plain text file that contains n+1n+1n+1 lines. The input graph has to be undirected, without self-loops and without parallel edges. The first line contains two integers

n mn\text{ }mn m

The remaining nnn lines store information about the actual structure of the graph. In particular, the iii-th line contains information about the iii-th vertex viv_ivi​, in the form of

wi ui1 ui2 … uikw_i \text{ } u_{i1} \text{ } u_{i2} \text{ } \dots \text{ }u_{ik}wi​ ui1​ ui2​ … uik​

where wiw_iwi​ is the weight associated with viv_ivi​ and ui1…uiku_{i1}\dots u_{ik}ui1​…uik​ are the neighbors of viv_ivi​.

Output File Format

There are two kinds of output files depending on the algorithm types we choose. If we only reduce the graph, the output is a graph obtained from the input by reductions. The output file contains n+2n+2n+2 lines, where nnn is the number of vertices in the output. The first n+1n+1n+1 lines has same format wiith the input file. The last line contains two integers

w1 w2w_1 \text{ }w_2w1​ w2​

where w1w_1w1​ is the weight got by reductions and w2w_2w2​ is the total weight of the output.

Otherwise, we run the whole algorithm. Now, the output is an independent set found by the algorithm. The set is stored in a plain text that contains two lines. The first line contains two integer

k w k\text{ } w k w

where kkk is the size of the set and www is the total weight of the set. The second line contains the vertices of the set in the form of

u1 u2 … uk u_1 \text{ } u_{2} \text{ } \dots \text{ }u_{k}u1​ u2​ … uk​

3. Algorithmic Recap

Our algorithm is based on a branch-and-reduce framework, where reductions and branches are excuted iteratively. Following previous works (refs to be added), we design some powerful reductions to improve the experimental performence significantly.

In reality, almost all graphs follow power-law distribution. It means almost graphs are sparse since there are many low-degree vertices. When solving MIS, a important reason that the exsiting algorithm can runs efficiently is that low-degree verices can be reduced, especially degree-2 vertices (all can be reduced). However, most reductions for MIS can not hold for MWIS, including the low-degree reductions.

Here, we introduce some new reductions that can works for low-degree graph structure. Almost all our reductions can be explained by a meta-reduction, named by unconfined reduction.

Unconfined Reduction. Let SSS be a set contained in all maximum weighted independent set. Then, for any independent set S′⊆N(S)S'\subseteq N(S)S′⊆N(S), it holds that

w(S′)<w(S∩N(S′))+w(S′′),w(S')<w(S\cap N(S'))+w(S''),w(S′)<w(S∩N(S′))+w(S′′),

where S′′S''S′′ is some independent set of G[N(S′)∖N[S]]G[N(S')\setminus N[S]]G[N(S′)∖N[S]]. Furthermore, if S'' is unique, then S∪S′′S\cup S''S∪S′′ is must contained in all maximum weighted independent set.

The first condition is called termination and the second one is named by expansion. Use this two condition, for any vertex vvv, either vvv is terminated when there is a maximum weighte indepdent set not containing vvv or finding a set SvS_vSv​ confining vvv by expansion. The set SvS_vSv​ is said to be confining vvv if there is a maximum weighted independent either containing SvS_vSv​ or not containing vvv.

This reduction is too complex to apply in practice. So, our reuctions are obtained from this by relaxtion and some special cases. For more details, you can read the paper to be published.
